# Nietzsche Research Model

This document describes the Phase 2.5 research structure used by the Nietzsche prototype.

## Work Metadata

Generated by:

```powershell
python .\scripts\build_nietzsche_metadata.py
```

Output:

```text
data/nietzsche_metadata.json
```

Each `works.<work_id>` record stores:

- `author`, `author_id`, `work_id`
- `title`, `korean_title`, `date_label`
- `category_id`, `category_title`
- `source_path`, `work_url`, `source_url`
- `language`, `edition_note`
- `heading_count`, `paragraph_count`, `char_count`
- `sha256`
- `concept_ids`

## Citation

The work page builds citation text from the current URL hash.

Whole work:

```text
Friedrich Nietzsche, Morgenröthe (M). Personal Archive of Literature. http://127.0.0.1:8793/work/nietzsche/M
```

Anchored section:

```text
Friedrich Nietzsche, Morgenröthe (M), § 1. Personal Archive of Literature. http://127.0.0.1:8793/work/nietzsche/M#sec-1
```

Anchored paragraph:

```text
Friedrich Nietzsche, Morgenröthe (M), paragraph text. Personal Archive of Literature. http://127.0.0.1:8793/work/nietzsche/M#p-0001
```

The citation format is intentionally simple for now; Phase 4 can add CSL/Zotero export.

## Notes

Notes are stored as JSONL records in:

```text
data/notes/nietzsche_notes.jsonl
```

Record shape:

```json
{
  "id": "hex uuid",
  "created_at": "local ISO timestamp",
  "author": "nietzsche",
  "work_id": "M",
  "target_id": "sec-1",
  "target_label": "§ 1",
  "quote": "optional selected text",
  "note": "personal note text",
  "tags": ["morality", "genealogy"],
  "url": "/work/nietzsche/M#sec-1"
}
```

The source Markdown files are not modified.

## Concepts

Concept seeds live in:

```text
data/nietzsche_concepts.json
```

The work page shows concepts whose `works` list contains the current `work_id`.
